\section{Evaluation}
We run the TÜLU-3 dev smoke test with \texttt{run\_tulu3\_dev\_limit100.sh}, dispatching 11 validation suites across the local A6000 fleet and constraining each task to a 100-example slice (MMLU uses 100 questions per subject, yielding 5{,}700 evaluations). This regimen provides a fast signal on diverse reasoning, coding, alignment, and factuality workloads while keeping GPU time manageable.

\section{Data Filtering}
We label every supervised (SFT), preference (DPO), and RLVR sample with the minimum calendar year consistent with its question--answer bundle. The latest sweep (session \texttt{2026-01-05\_14-31PT}) processed 1,000 examples per corpus, enforcing a conservative policy that forces the most recent referenced year to dominate. Figures~\ref{fig:sft}--\ref{fig:rlvr} show both year and category distributions for each dataset family; these plots make it easy to confirm that earlier buckets remain uncontaminated while still exposing modern content when desired.
